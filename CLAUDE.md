# Claude를 위한 프로젝트 가이드

이 문서는 Claude가 이 프로젝트에서 Git 작업을 수행할 때 따라야 할 규칙을 정리한 문서입니다.

## 브랜치 및 PR 규칙

### 브랜치 네이밍

- 형식: `ch{시작소제목번호}-{주제}`
- 예시: `ch1.5-requests-and-responses`
- 규칙: 여러 소제목을 함께 작업하더라도, 시작한 소제목 번호를 기준으로 브랜치명 생성

### 커밋 메시지

- **한글로 작성**
- 작업한 내용을 간결하게 요약
- 예시: `HTTP 요청/응답 처리 및 HTML 표시 기능 구현`

### PR 생성

- **제목 형식**: `Chapter {소제목}: {내용}`

  - 단일 소제목 예시: `Chapter 1.5 요청과 응답`
  - 여러 소제목 예시: `Chapter 1.5 요청과 응답 외` (시작 소제목 + '외')

- **본문 형식**:

  ```markdown
  ## 작업 내용

  - 1.5 요청과 응답
  - 1.6 HTML 표시하기

  ## 주요 변경사항

  - HTTP 요청/응답 파싱 구현
  - HTML 태그 제거 및 텍스트 출력 기능 추가
  ```

## 작업 프로세스

1. 새로운 챕터 작업 시작 시:

   - **중요**: `main` 브랜치로 이동 후 최신화
   - 최신 `main`에서 새 브랜치 생성
   - 브랜치명: 시작 소제목 기준

2. 작업 완료 시:

   - 변경사항 커밋
   - PR 생성 (여러 소제목 작업 시 '외' 붙이기)
   - PR 본문에 상세 내용 기재

3. PR 머지 후:
   - `main` 브랜치로 돌아가기
   - 다음 작업을 위한 준비

## 연습문제 작업 규칙

각 챕터 종료 후 연습문제를 진행할 때는 다음 규칙을 따릅니다:

### 브랜치 네이밍

- 형식: `ex{챕터}.{문제번호}-{주제}`
- 예시: `ex1.1-http11`, `ex1.7-redirects`

### PR 생성

- **제목 형식**: `Exercise {챕터}-{문제번호} {제목}`
  - 예시: `Exercise 1-1 HTTP/1.1 헤더 지원`
  - 콜론 없이 작성

- **본문 형식** (템플릿 없이 수동 작성):

  ```markdown
  ## 문제

  {원문 문제 내용}

  ## 핵심 개념

  - {주요 개념 1}
  - {주요 개념 2}

  ## 주요 변경사항

  ### 구현 코드

  - {변경사항 1}
  - {변경사항 2}

  ### 참고 문서

  - {문서명}: {설명} (필요시)
  ```

### 작업 방식

1. Claude가 추천하는 핵심 연습문제만 선별하여 진행
2. 각 문제마다 별도 브랜치 생성
3. Claude가 답안 코드 및 설명 제공
4. 사용자가 학습 후 빠르게 커밋/푸시하여 진행

## 예시

### 챕터 작업 예시

작업: 1.5 요청과 응답, 1.6 HTML 표시하기를 함께 진행

```bash
# main 브랜치 최신화
git checkout main
git pull origin main

# 브랜치 생성 (시작 소제목 기준)
git checkout -b ch1.5-requests-and-responses

# 작업 후 커밋 (한글로 작성)
git add .
git commit -m "HTTP 요청/응답 처리 및 HTML 표시 기능 구현"

# PR 생성 (제목과 본문 모두 한글, 여러 소제목이므로 '외' 붙임)
gh pr create --title "Chapter 1.5 요청과 응답 외" --body "..."
```

### 연습문제 작업 예시

작업: Exercise 1-1 (HTTP/1.1)

```bash
# main 브랜치 최신화
git checkout main
git pull origin main

# 연습문제 브랜치 생성
git checkout -b ex1.1-http11

# 작업 후 커밋
git add .
git commit -m "Exercise 1-1: HTTP/1.1 헤더 지원 추가"

# PR 생성
gh pr create --title "Exercise 1-1 HTTP/1.1 헤더 지원" --body "..."
```
