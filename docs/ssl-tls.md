# SSL vs TLS

## 간단한 요약

**TLS는 SSL의 후속 버전입니다.** 같은 목적(암호화 통신)을 하지만, TLS가 더 새롭고 안전합니다.

## 역사적 발전 과정

```
SSL 1.0 (1994) - 공개되지 않음 (보안 문제)
  ↓
SSL 2.0 (1995) - 넷스케이프가 개발, 보안 취약점 발견
  ↓
SSL 3.0 (1996) - 보안 개선
  ↓
TLS 1.0 (1999) - SSL 3.0 기반, IETF 표준화 (이름 변경)
  ↓
TLS 1.1 (2006) - 보안 강화
  ↓
TLS 1.2 (2008) - 현재 널리 사용
  ↓
TLS 1.3 (2018) - 최신, 더 빠르고 안전
```

## 주요 차이점

| 항목 | SSL | TLS |
|------|-----|-----|
| **개발** | 넷스케이프 | IETF 표준 단체 |
| **최신 버전** | SSL 3.0 (1996) | TLS 1.3 (2018) |
| **보안** | 취약점 많음 | 더 안전 |
| **현재 상태** | 사실상 폐기됨 | 현재 사용 중 |
| **성능** | 느림 | 더 빠름 |

## 실무에서는?

**용어는 혼용되지만, 실제로는 모두 TLS를 사용합니다:**

```python
# 이름은 ssl 모듈이지만
import ssl

# 실제로는 TLS를 사용
ctx = ssl.create_default_context()
# ↑ 내부적으로 TLS 1.2 이상 사용
```

- **HTTPS** = HTTP + TLS (SSL이 아님!)
- 사람들이 "SSL 인증서"라고 부르지만 실제로는 "TLS 인증서"
- Python의 `ssl` 모듈은 실제로 TLS를 구현

## 왜 여전히 "SSL"이라고 부르나?

1. **역사적 이유**: SSL이 먼저 나와서 이름이 널리 퍼짐
2. **브랜드 인지도**: "SSL"이 더 익숙함
3. **Python 모듈명**: `ssl`이 그대로 유지됨

## 현재 상황

```
실제 사용되는 것:
- TLS 1.2 ✅ (대부분)
- TLS 1.3 ✅ (최신)

더 이상 사용 안 함:
- SSL 2.0 ❌
- SSL 3.0 ❌
- TLS 1.0 ❌ (2020년부터 주요 브라우저에서 지원 중단)
- TLS 1.1 ❌ (2020년부터 주요 브라우저에서 지원 중단)
```

## 코드에서 확인하기

```python
import ssl

# Python이 지원하는 프로토콜 확인
print(ssl.PROTOCOL_TLS)  # TLS를 사용
print(ssl.PROTOCOL_TLSv1_2)  # TLS 1.2
print(ssl.PROTOCOL_TLSv1_3)  # TLS 1.3

# create_default_context()는 자동으로 안전한 TLS 버전 선택
ctx = ssl.create_default_context()
```

## 결론

- **SSL**: 1990년대 암호화 프로토콜 (구식, 폐기됨)
- **TLS**: SSL의 개선된 후속 버전 (현재 사용)
- 용어는 "SSL"이 흔하지만, **실제로는 모두 TLS 사용**
- `ssl.create_default_context()`는 안전한 TLS 버전만 사용함

간단히 말하면: **SSL = 구버전, TLS = 신버전, 둘 다 같은 목적이지만 TLS가 더 안전**합니다!
